# Security Policy

## Authorized & Responsible Use

`hwatlib` is offensive-security tooling intended for **legitimate, authorized**
use only: penetration testing engagements with written permission, Capture The
Flag (CTF) competitions, security research on systems you own, and educational
or defensive purposes.

By using this software you agree that:

- You will only scan, probe, exploit, or otherwise interact with systems for
  which you have **explicit, prior, written authorization** (or that you own).
- You are solely responsible for complying with all applicable local, national,
  and international laws and regulations.
- The authors and contributors accept **no liability** for misuse or for any
  damage caused by this software. It is provided "as is" under the terms of the
  [MIT License](LICENSE).

Unauthorized access to computer systems is illegal in most jurisdictions. If you
are unsure whether your use is authorized, **do not run this software.**

## Supported Versions

Security fixes are applied to the latest released minor version on the `main`
branch. Older versions are not maintained.

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability **in hwatlib itself** (for example, an
issue that could harm an operator running the tool), please report it privately
rather than opening a public issue.

- Preferred: open a [GitHub Security Advisory](https://github.com/iabdullah215/hwatlib/security/advisories/new).
- Alternatively, email the maintainer at **muhammadabdullah8040@gmail.com**
  with the subject line `hwatlib security`.

Please include:

- A description of the issue and its impact.
- Steps to reproduce or a proof of concept.
- Any suggested remediation.

You can expect an initial acknowledgement within **7 days**. Please give us a
reasonable window to release a fix before any public disclosure.
